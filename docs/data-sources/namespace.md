# context_namespace (Data Source)

The `context_namespace` data source appends a named namespace entry to the context stack.  Namespaces represent organisational layers such as environment, region, or team.  Stack multiple `context_namespace` calls to build a hierarchy; pass each one's `context` output as the next one's `context` input.

The `id_casing`, `id_prefix`, and `include_resource_type_in_id` attributes shape how `context_label` later builds the resource ID.  When any of these are set on a namespace, the namespace-level value takes precedence over the provider-level default.  If multiple namespaces set the same attribute, the **last** one in the stack (deepest leaf) wins.

Mapper functions declared on a namespace are applied bottom-up (deepest namespace first) when `context_label` evaluates the stack.

## Example Usage

```terraform
terraform {
  required_providers {
    context = {
      source = "snapsoft-io/context"
    }
  }
}

provider "context" {}

# ── Basic: start a new context stack ─────────────────────────────────────────

data "context_namespace" "env" {
  name    = "production"
  context = { stack = [] }
}

# ── Stack a second namespace on top of the first ──────────────────────────────

data "context_namespace" "region" {
  name    = "eu-west-1"
  context = data.context_namespace.env.context
}

# ── id_prefix: prepend a fixed string to all IDs produced downstream ──────────

data "context_namespace" "env_with_prefix" {
  name      = "production"
  id_prefix = "acme"
  context   = { stack = [] }
}

# ── id_casing: control the casing style of the generated ID ──────────────────
# Accepted values: "kebab-case" (default), "snake_case", "camelCase", "PascalCase".

data "context_namespace" "env_snake" {
  name      = "production"
  id_casing = "snake_case"
  context   = { stack = [] }
}

data "context_namespace" "env_pascal" {
  name      = "production"
  id_casing = "PascalCase"
  context   = { stack = [] }
}

# ── include_resource_type_in_id: append the resource_type to the ID ───────────

data "context_namespace" "env_with_rtype" {
  name                     = "production"
  include_resource_type_in_id = true
  context                  = { stack = [] }
}

# ── Combining all three ID-shaping attributes ─────────────────────────────────

data "context_namespace" "env_full" {
  name                     = "staging"
  id_prefix                = "acme"
  id_casing                = "snake_case"
  include_resource_type_in_id = true
  context                  = { stack = [] }
}

# ── vars: namespace-scoped variables, available to mappers at evaluation time ──

data "context_namespace" "env_with_vars" {
  name    = "production"
  context = { stack = [] }
  vars = {
    cost_center = "cc-42"
    owner       = "platform-team"
  }
}

# ── mappers: JQ expressions applied when context_label evaluates the stack ────

data "context_namespace" "env_with_mappers" {
  name    = "production"
  context = { stack = [] }
  vars = {
    environment = "production"
  }
  mappers = [
    {
      name     = "Tag with environment"
      function = "(.context.outputs.tags += {Environment: .vars.environment}).context"
    },
    {
      name          = "Tag with cost centre (non-production only)"
      run_condition = ".vars.environment != \"production\""
      function      = "(.context.outputs.tags += {CostCentre: \"sandbox\"}).context"
    }
  ]
}
```

## Schema

### Required

- `context` (Attributes) The current context stack. Pass `{ stack = [] }` to start a new stack, or pass the `context` output of a previous `context_namespace` to extend an existing one. (see [below for nested schema](#nestedatt--context))
- `name` (String) The name of this namespace entry. Becomes part of the generated resource ID.

### Optional

- `id_casing` (String) Casing style for generated IDs. One of: `kebab-case` (default), `snake_case`, `camelCase`, `PascalCase`. Overrides the provider-level default. The value from the deepest namespace in the stack wins.
- `id_prefix` (String) Fixed string prepended to the generated ID. Overrides the provider-level default. The value from the deepest namespace in the stack wins.
- `include_resource_type_in_id` (Boolean) When `true`, the `resource_type` value from the downstream `context_label` is appended to the generated ID. Overrides the provider-level default. The value from the deepest namespace in the stack wins.
- `mappers` (Attributes List) JQ-based mapper functions attached to this namespace. Applied bottom-up when `context_label` runs. (see [below for nested schema](#nestedatt--mappers))
- `vars` (Map of String) Variables scoped to this namespace. Available as `.vars` inside mapper functions. Override provider-level vars on key conflicts.

### Read-Only

- `context` (Attributes) The updated context stack with this namespace appended. Pass this to the next `context_namespace` or to a `context_label`. (see [below for nested schema](#nestedatt--context))
